☠️ F Tier APIs

These APIs are deprecated, obsolete, or widely considered failures. You should avoid them in new projects, but it’s good to recognize them when you see old code.

Application Cache (AppCache)

Used to make web apps work offline. Replaced by Service Workers.

How it looked:

<html manifest="app.appcache">
<!-- The manifest file listed offline resources -->
⚠️ Deprecated because it was buggy, insecure, and had poor design. ✅ Use Service Workers + Cache API instead.

WebSQL Database

A browser-based SQL database API. Replaced by IndexedDB.

How it looked:

const db = openDatabase("mydb", "1.0", "Test DB", 2 * 1024 * 1024);
db.transaction(tx => {
  tx.executeSql("CREATE TABLE IF NOT EXISTS users (id, name)");
});
⚠️ Deprecated due to lack of standardization. ✅ Use IndexedDB or localStorage instead.

Battery Status API

Exposed battery charge and charging status. Removed for privacy reasons.

How it looked:

navigator.getBattery().then(battery => {
  console.log("Battery level:", battery.level * 100 + "%");
});
⚠️ Deprecated because websites used it to fingerprint users. ✅ Avoid — browsers don’t support this anymore.

Old Gamepad Extensions

The Gamepad API still exists, but older extensions and vendor-specific hacks are gone.

Modern version:

window.addEventListener("gamepadconnected", e => {
  console.log("Gamepad connected:", e.gamepad.id);
});
⚠️ Avoid old prefixed APIs (`webkitGamepad`, `mozGamepad`). ✅ Use the standardized Gamepad API.
